School TimeTable
Generator

An automated timetable builder for schools and colleges that reduces manual planning, avoids teacher clashes, respects lunch breaks, and produces print-ready section-wise schedules.

PHP MySQL HTML CSS

The Problem
Why the project matters

Manual timetable planning creates avoidable chaos

Too much manual work

Building schedules by hand takes hours, especially when periods, teachers, and sections need to line up across the full week.

Multiple sections multiply the effort

When a class has sections like 10-A, 10-B, and 10-C, each needs a separate valid timetable with shared staff constraints.

Teacher clashes happen easily

The same teacher can accidentally be assigned to more than one section in the same slot unless conflict checks are built into the system.

Existing tools are often complex

Many solutions are expensive, installation-heavy, or not designed for a simple web-based school workflow.

Features
Core product highlights

What the system can do

Multiple classes and sections

Handles multiple class groups with independent section-wise timetable generation.

Automatic subject distribution

Uses subject quotas to spread weekly periods in a balanced way rather than clustering them randomly.

Teacher override per section

Supports section-level teacher changes when one section needs a different faculty assignment for the same subject.

Lunch break aware

Respects lunch timing so the generated timetable reflects real working constraints.

Print and PDF ready

Produces output that can be presented, printed, or exported cleanly for school use.

Workflow
From form to final output

How the timetable gets generated

1

Collect input

User enters school timing, classes, subjects, and teacher mapping.

2

Build slots

The backend converts school hours into ordered period slots and inserts lunch at the correct time.

3

Assign subjects

A greedy scheduling pass selects valid candidates based on remaining quota and conflicts.

4

Render sections

Each section gets its own timetable view with subject, teacher, lunch, and free period states.

5

Share output

The final layout is ready for browser viewing, printing, or PDF export.

buildSlots()

  • Creates a sequence of time slots between school start and end.
  • Inserts the lunch break as a special slot so it is visible in the final timetable.

generateTimetable()

  • Loops through days, classes, and slots in a consistent order.
  • Chooses the strongest remaining valid candidate at each step.
Algorithm
Greedy scheduling logic

The core assignment strategy

// For each day, class, and time slot:
foreach ($days as $day) {
  foreach ($classes as $class) {
    foreach ($slots as $slot) {
      $candidates = findCandidates($subjects, $booked, $slot);
      // Sort by remaining quota so important subjects get priority
      usort($candidates, fn($a, $b) => $b['quota'] - $a['quota']);
      $selected = $candidates[0] ?? null;
      if (!$selected) { continue; }
      $quota[$class][$selected['subject']]--;
      $booked[$day][$slot][$selected['teacher']] = true;
    }
  }
}

Conflict prevention

The booking matrix records teacher usage per day and per slot so the same teacher cannot be assigned twice at the same moment.

Graceful fallback

If no valid candidate exists, the timetable can safely leave that slot as a free period instead of failing.

Database
Relational model overview

MySQL schema for session-based data

tt_sessions

  • Core fields: school name, start time, end time
  • Planning fields: period duration, lunch time, days per week

tt_subjects

  • Links: subject belongs to a session
  • Rules: stores periods per week for quota balancing

tt_teachers

  • Faculty info: teacher name and mapped subject
  • Availability: lunch start and lunch end support

tt_timetable

  • Output rows: class, day, period number, start and end
  • State flags: lunch and free-period markers for rendering

Why this structure works

The schema keeps inputs, constraints, and generated timetable rows separated, making the generator easier to maintain and extend.

Integrity support

Foreign keys and cascade behavior help keep session-scoped records consistent when a school setup is updated or removed.

What users provide

What the user enters first

Classes and sections

Examples like Class 10 with sections A, B, and C allow the system to create parallel but independent section timetables.

Subjects and default teachers

Each subject is linked to a teacher and a weekly period quota so the generator knows both ownership and demand.

Optional overrides

If one section needs a different teacher for a subject, the override is stored without changing the default for other sections.
